Title: 做了一个彩票 chatbot，试运营一周总结
Date: 2018-05-17
Category: NLP
Tags: 炼丹, 挖坑
Summary: 基于 chatbot 的 UX，一定是有未来的，会成为下一代主流 UI。用了一周彩票的机器人，我已经不习惯网页版的查询工具了。


通过微信购买彩票的用户，越来越多。发红包给彩票店老板，直接下注出票。
同时，彩民还需要查询历史开奖信息，做一些分析，也想找人讨论一下。
如果有一个 chatbot 能实时回复准确数据，推送关键线索，非常有想象空间。

chatbot 这个方向，大家都想做，但做的用户体验都不太理想。
一两年前，调研过 chatbot 市场，甚至得出了 chatbot 不靠谱的结论。

当时最流行的 demo 是点餐机器人，一顿饭点下来，心力憔悴，不会再爱。
来来回回十几个流程，无数的问题／选项。
远不如给一页面，手指点一点省事。

其实，这些机器人，技术本身是很好的。
只是，太工程师 style，没有贴近运营／售后人员的日常需求。

以点披萨为例，我想，大多数人会直接发一条消息： "九寸芝士披萨，微辣，少盐。"
而不是需要一个“智能”机器人陪聊多个回合：你想要几寸？什么辣度？多盐少盐？

chatbot 与 GUI UX 的区别，恰如 Linux 和 windows。
打开对话框，发个 command 就搞定了。
command 的个性化程度／定制性远超过 GUI UX。

很简答的道理，GUI 的 UX 都需要导航栏。
打开美团 APP，第一步是选择点外卖，还是打车，然后再搜索。
chatbot，打开以后，直接发送搜索语句，一句话搞定。

基于 chatbot 的 UX，一定是有未来的，会成为下一代主流 UI。
用了一周彩票的机器人，我已经不习惯网页版的查询工具了。

常用功能，没有什么是一个 command 搞不定的，
如果有，那就发 2 个 command。

chatbot 这个名字，其实很有误导性。
叫 natural language command parser 更准确一些。
名字换了以后，对应的技术方案也就明确了 -- 规则引擎。

我甚至在想，
国内的实时聊天系统，有机会出现一个能对抗微信的，那就是支持 bot 接入。
早期，主要针对程序员／金融等白领群体。
slack，telegram 都是这个趋势，只是他们在其他方面不适合中国国情。

技术挑战，自然也是存在的，主要是短会话管理。
比如，很多心急的用户，会快速连发三个消息，如何有效拼接成一个 command？

```bash
> 九寸芝士披萨
> 微辣
> 少盐
```

最后，彩票机器人，运营效果如何呢？
做了一个星期，单日销量达到了一家普通彩票店的水平。

然后，被微信屏蔽了。
