# System Design Overview

## 总体定位

1. 个人 AI idea 的 demo 演示
2. 沉淀 AI 入门的基础知识、论文、代码、数据集

## 素材

#### CV 类

1. 基础的人脸检测，画框、landmarks、属性识别、特征提取
2. 人脸识别。流量网红，维秘模特，porn star
3. 图片鉴黄。
4. 以图搜图。小姐姐搜索器。
5. TODO OCR 数字验证码识别
6. TODO OCR 场景文字识别/车牌识别
7. TODO 自动带口罩、化妆、美图。
8. TODO 人体识别。姿势、动作分类。
9. TODO 人体跟踪。

模版要求

输入：

1. 图片/视频。3 个来源：摄像头、本地上传、网络 URL
2. 场景 & 模型名称。比如：人脸识别、人体识别
3. 模型参数。底库名称、阈值等。

输出：不同场景，展示方式有区别

1. 对象识类。对象名称，位置坐标，标注文本
2. 属性识别类。属性的 key：value 字典
3. 搜索类。命中的结果数组。
4. TODO 跟踪类。

网站功能补充：

1. 用户反馈，主要是针对识别结果的正确性。
2. 标注网站。从 tag 标注做起。

## 架构设计

分数据、后端、前端 3 层

#### 数据层 DataPipeline

功能定位

1. 模型仓库
2. 数据集管理。大量图片存储
3. 大型计算任务。主要是模型推理。

#### 后端层 ApiServer

功能定位

1. 用户 UI 和模型之间的胶水。提供 API
2. 较轻的实时计算。主要是搜索，数据格式改写与重组
